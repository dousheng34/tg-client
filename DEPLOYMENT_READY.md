## 🌐 Опции развертывания

### Вариант 1: Render.com (РЕКОМЕНДУЕТСЯ - Бесплатно)

1. Перейдите на [render.com](https://render.com)
2. Создайте аккаунт
3. Нажмите "New +" → "Web Service"
4. Подключите GitHub репозиторий
5. Заполните:
   - **Name:** `kazakh-literature-bot`
   - **Runtime:** `Python 3`
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `python bot.py`
6. Добавьте переменные окружения:
   - `TOKEN` = `8167312154:AAFNFQqJ7_dwbbqFjlO4tTbpGSkD9lQP4rM`
   - `BOT_ID` = `7578426218`
7. Нажмите "Create Web Service"

✅ Бот будет работать 24/7!

### Вариант 2: DigitalOcean ($5/месяц)

1. Создайте аккаунт на [digitalocean.com](https://digitalocean.com)
2. Создайте новый Droplet (Ubuntu 22.04)
3. SSH в сервер:
   ```bash
   ssh root@YOUR_SERVER_IP
   ```
4. Установите Python и Git:
   ```bash
   apt update && apt install -y python3 python3-pip git
   ```
5. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/YOUR_USERNAME/tg-client.git
   cd tg-client
   ```
6. Установите зависимости:
   ```bash
   pip3 install -r requirements.txt
   ```
7. Создайте `.env` файл:
   ```bash
   nano .env
   ```
   Вставьте:
   ```
   TOKEN=8167312154:AAFNFQqJ7_dwbbqFjlO4tTbpGSkD9lQP4rM
   BOT_ID=7578426218
   MODE=polling
   LOG_LEVEL=INFO
   ```
8. Запустите бота в фоне:
   ```bash
   nohup python3 bot.py > bot.log 2>&1 &
   ```

✅ Бот будет работать 24/7!

## 🆘 Решение проблем

### Бот не запускается
```bash
# Проверьте токен в .env
cat .env

# Проверьте зависимости
pip install -r requirements.txt

# Запустите с логами
python bot.py
```

### Ошибка "TOKEN не найден"
- Убедитесь, что `.env` находится в той же папке, что и `bot.py`
- Проверьте, что `TOKEN` правильно скопирован

### Бот не отвечает на команды
- Убедитесь, что бот запущен
- Проверьте логи: `tail -f bot.log`
- Перезагрузите бота: `Ctrl+C` и `python bot.py`
